# 安科剧本 JSON 生成提示词

## 你的任务

我将给你一段帖子内容（通常来自论坛、贴吧或社交媒体），内容描述了某个 FFXIV 固定队或角色的经历、属性、故事。

你需要：
1. **阅读帖子内容**，提取其中出现的可随机化要素（如性别、职业、水平、性格、经历等）
2. **修改或补充** 下方提供的现有 JSON 节点（修复 `nextNode: null` 的断裂分支，补充缺失节点）
3. **生成新的节点**，以 JSON 数组格式输出，可直接合并到现有文件
4. **主线原则**：严格按照帖子中实际骰出的结果作为唯一有效分支（`nextNode` 有值），其他所有 outcome 的 `nextNode` 一律为 `null`。两条分支汇总到同一节点的情况**暂不处理**，留待后续补全。
5. 如果有不确定的地方提出问题，不急着修改文件
6. **禁止自行推断帖子楼层内容**，如果帖子数据无法获取，中断流程并告知我
7. **每次输出20个node**，当我说**继续**时再获取后续内容
8. **每次修改 `chapter1.json` 后，必须同步更新本文件末尾的进度记录和角色状态快照**

## 数据结构说明

每个节点是一个 `ScriptNode` 对象，完整结构如下：

```json
{
  "id": "chapter1_nodeN",
  "title": "节点标题",
  "content": "展示给玩家的文案\n换行用 \\n，最后一行通常以冒号结尾作为判定提示",
  "type": "dice",
  "choices": [
    {
      "text": "判定项描述",
      "diceRequired": 6,
      "outcomes": [
        {
          "match": 1,
          "optionText": "选项文案（判定表里显示）",
          "resultText": "判定结果文案（历史记录里显示）",
          "nextNode": "chapter1_nodeN"
        }
      ]
    }
  ]
}
```

### 关键规则

| 字段 | 规则 |
|------|------|
| `id` | 全局唯一，格式 `chapter{章节号}_node{序号}`，序号从现有最大值续接 |
| `match` | outcomes 中所有 match 必须**连续且无重叠**地覆盖 `1 ~ diceRequired` |
| `diceRequired` | 等于所有有效 outcomes 中 match 最大值（`null` 分支不算） |
| `nextNode` | **仅原帖中实际出现的骰点结果对应分支**才填真实节点 ID；其余所有分支**必须填 `null`** |
| `content` | 文案口吻统一为旁白/主持人视角，轻松幽默，符合 FFXIV 二次元社区风格 |
| `resultText` | 历史记录展示文案；**若帖主在骰点结果后紧跟输出了描述文字，则必须严格照抄原文，不得改字、省字或调整标点** |

### 节点粒度规则

- **每次骰点事件独立成一个节点**；纯叙事内容（无骰点的楼层）合并进相邻节点的 `content` 或 `resultText`
- **一楼中若存在多次骰点，拆分为多个节点**，用 `nextNode` 串联

## 🎲 熟练度设计说明

JSON 中各角色的骰点基础值（如 `黑骑：110+d100`）是**写死在 `resultText` 里的叙事文本**，忠实还原帖子当时的状态，不是运行时动态变量。

当前系统为**忠实重演**模式：玩家重走帖主实际骰出的路径，角色熟练度变化只是叙事内容，不影响判定逻辑。若未来需要「真随机成长」模式，应作为新剧本类型单独实现，不改现有数据结构。

**因此：每次新增节点时，须参考下方角色状态快照中的当前基础值，确保 `resultText` 中的数值与帖子一致。**

## 输出格式要求

- 只输出 **JSON 数组**，不加任何解释文字
- 数组中每个元素是一个完整的 `ScriptNode` 对象
- 如需修改现有节点，输出修改后的完整节点对象，并在最前面加注释说明
- 所有字符串使用双引号，JSON 合法，可直接粘贴
